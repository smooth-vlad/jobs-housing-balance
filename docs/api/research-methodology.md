# API Research Methodology

## Методология исследования
1. **Официальная документация Colossal Order** - ПРИОРИТЕТНЫЙ источник
2. **Проверка компиляции** - ошибки показывают несуществующие свойства
3. **Поиск в интернете** - конкретные запросы, примеры из модов
4. **Анализ существующих модов** - GitHub, Steam Workshop
5. **Тестирование** - проверка через компиляцию
6. **Документирование** - запись найденных решений

## Источники информации
- **Simtropolis Community**: UI компоненты и примеры
- **Cities: Skylines Modding Guide**: официальная документация
- **Steam Workshop**: реальные моды
- **GitHub**: открытый код модов

## Правильный подход
- ✅ **НАЧИНАТЬ с официальной документации Colossal Order**
- ✅ Исследование API через интернет перед написанием кода
- ✅ Изучение примеров из реальных модов
- ✅ Проверка доступных свойств через компиляцию
- ✅ Использование `Find<T>()` для доступа к внутренним компонентам
- ✅ Проверка на `null` перед использованием

## Частые ошибки
- ❌ **Игнорирование официальной документации Colossal Order**
- ❌ Написание кода без исследования API
- ❌ Использование несуществующих свойств
- ❌ Отсутствие проверки на `null`
- ❌ Игнорирование примеров из реальных модов
